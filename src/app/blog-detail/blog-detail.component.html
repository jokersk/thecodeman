萬事起頭難

搞左成日，終於搞掂個site。 終於有自己個site啦。為左唔悲慘既歷史再一次重演。 我決定寫低我到底係點樣set up呢個site，等第日自己可以睇翻。

1. 買server

講起慚愧，雖然我做左IT狗多年，但係搭建server都係未試過。自己幻想有自己website好多年，但係因為懶癌，遲遲都唔做。 如果唔係今次真系火燒眼眉，睇來自己都係唔會做。
咁一開始梗係揀好出名既a字頭公司啦，於是我就註冊左個account，諗住大展身手啦。點知，入面D野超級複雜，對於咩都唔識既人來講簡直噩夢 ，研究左兩個就放棄左（我仲將自己信用卡綁左落去，遲D記得remove個account）。 然後問google大神咩server好，於是就搵左另一間i字頭公司， 超級平咁買左個host，set up埋domain。以為今次冇死。又點知！搵來搵去冇得ssh入去裝野。咁我就問cs點樣ssh入去啦，一問，先知道原來我個係share host， 唔係vps。原來要vps先可以ssh，一開始我真系唔知咩係vps，次次都係人地幫我set server。
咁之後就要個cs幫我remove account 同退錢，但係到宜家（2018-2-15）都仲未remove account（遲D記得跟跟佢）
又過一日，我又再問google大神，有咩vps hosting好，於是就搵左一件v字頭公司。 終於買左個vps翻來，以為所有野都會好順利，點知，呢度先系噩夢開始。

2. set server

我買既server系一個 Ubuntu 系統， 而Ubuntu系一個操作系統。 Ubuntu本身系非洲南部一種語言一詞，意思系“我存在因為大家的存在”。 因為細個受《獅子王》影響，對非洲詞好有好感，於是我選擇Ubuntu。 於是乎，我手上就有一個os，同一個ip address。
我滿懷期待系broswer入個ip address睇下有咩事發生。 結果系咩事都冇發生。 於是乎根據我既經驗，裝左個php先：

sudo apt-get install php

佢自動咁，同我裝左 PHP 7.1 （最新版）， 跟住我就touch一個php file，再行

php -S localhost:4000

再系broswer 入 ip_address:4000 , 激動！ 果然成功。

系呢度解釋下 php -S localhost:4000 系咩。 如果你系自己部電腦創建一個html file，你自己睇，你只要double click，佢就可以睇到。 但如果想用手機都睇到呢個file，咁點做呢？ 我地就要set一個web伺服器，令到電話可以用broswer睇到個html 文件。 而php -S xxx 就係一個php 內建 的一個伺服器。雖然方便，都係性能不好。

我需要一個真正的伺服器。 我選擇nginx。 因為 nginx 比 apache 快同新。
安裝 nginx :

sudo apt-get install NGINX
